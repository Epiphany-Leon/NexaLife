### What's New

#### AI Woven Throughout Every Module

v0.2.0 is the first release where AI assistance is functional end-to-end across the full workspace — not just a settings toggle.

- **Quick Capture — smart module routing**: type any thought and the AI suggests the right module (Inbox / Execution / Lifestyle / Knowledge / Vitals) in real time, with a local keyword engine as fallback when no API key is configured.
- **Financial entries — AI category suggestion**: open any transaction detail and the AI reads the title, notes, and amount, then suggests a category from the catalog (e.g. "Subscription", "Dining", "Transport"). A one-tap chip applies the suggestion instantly.
- **Task metadata — AI auto-fill**: opening a task triggers automatic inference of category, tags, and project name. Hit the wand button to re-run at any time.
- **Relationship records**: AI generates interaction strategy, importance score (1–5), and next-action suggestion for each connection.
- **Notes and Vitals entries**: AI can generate a summary report for any note or vitals record.
- **Dashboard — AI Mentor panel**: the dashboard surfaces pattern recognition and weekly guidance based on your recent records. The floating AI Mentor button is now always visible — it guides you to Settings if not yet configured, or opens a direct chat window once an API key is in place.

#### Execution Module — Project Experience Upgrade

- **Project row redesign**: each project row now shows two coloured tag pills on the left — `[Short / Medium / Long]` horizon and `[Not Started / In Progress / Finished / Paused]` status — with edit and delete actions cleanly isolated on the right.
- **Project status — Paused added**: `ProjectStatus` gains a `.paused` case, so projects can now be flagged as suspended without being deleted. Summary pills in the Projects panel reflect live counts.
- **Project field as Combobox**: the project field in Task Detail is now a unified combobox — type a new project name directly, or click the `⌃⌄` arrow to pick from existing projects. AI auto-fills the field when enough task context is available.

#### AI Reliability Fixes

- **Module classification robustness**: `classifyText()` now uses 60 max tokens (was 20) and parses the AI response with a case-insensitive scan rather than an exact string match — so responses like "lifestyle" or "It's **Lifestyle**." are handled correctly.
- **Expanded offline keyword engine**: the local fallback for Lifestyle now covers `subscription / membership / fee / 扣费 / 会员 / 订阅 / 续费` and 10 more terms, so common spending captures route correctly even without an API key.
- **Financial AI validation**: the AI-suggested category is validated against the known catalog before being shown; partial matches (e.g. "Subscription service" → "Subscription") are resolved gracefully.

#### Storage and Settings

- **API token now stored as a local file** (previously Keychain): the token lives at `~/Library/Application Support/NexaLife/ai_api_token.txt` and is auto-migrated from Keychain on first launch. The file path is shown in Settings → AI.
- **Cherry Studio-style model picker**: a two-pane sheet (provider list on the left, model list on the right) replaces the flat dropdown. Custom providers can be added and removed with Terminal Preferences-style `+`/`−` controls.

#### Onboarding

- New **AI Mentor step** in the onboarding flow — configure your API key during first launch or skip and configure later.
- Language toggle (Auto / 中 / EN) available directly on the welcome screen.
- Three-step progress indicator: Choose mode → Create profile → AI Mentor.

#### Architecture

- `ContentView` reduced from 620 to 127 lines; module routing extracted to `ModuleWorkspaceRouter`.
- `VitalsView` reduced from 997 to 481 lines; `ExecutionView` from 1431 to 710 lines.
- `WorkspaceTheme` design system consolidated; all module views share a single token set.
- Docs folder moved out of the app bundle — app size reduced from ~30 MB to ~1.9 MB.
- Xcode Build Phase script permanently clears `com.apple.quarantine` xattr before every build, resolving recurring CodeSign failures.

### Bug Fixes

| # | Area | Description |
|---|------|-------------|
| 1 | AI | Quick Capture suggested "Inbox" for all spending-related text due to `maxTokens: 20` truncation and missing keywords. Fixed. |
| 2 | AI | Financial item detail showed "收件箱归类" as category (inherited from wrong AI routing). Root cause fixed in item 1; new AI categorization added to detail view. |
| 3 | AI | Connection test always returned generic failure when using DeepSeek v4-pro (thinking model consumed all 20 tokens for reasoning, leaving empty content). Fixed: `maxTokens` raised to 512, HTTP 200 always treated as success. |
| 4 | Execution | Creating a new project did not refresh the list or auto-select the new row. Fixed with `@State` + `FetchDescriptor` manual reload + `skipNextSelectionReset` sentinel. |
| 5 | Build | CodeSign failure on every build due to `com.apple.quarantine` xattr on bundled Markdown files. Fixed by moving Docs out of bundle and adding a Build Phase xattr-clear script. |
| 6 | Settings | "Open Settings" button in the AI chat sheet did not work (`NSApp.sendAction` path broken). Fixed with `@Environment(\.openSettings)` + `NSApp.activate`. |

### 新功能

#### AI 贯穿全模块

v0.2.0 是 AI 功能真正端到端覆盖整个工作台的第一个版本——而不只是一个设置开关。

- **快速捕捉 — 智能模块路由**：输入任何内容，AI 实时建议最适合的归属模块（收件箱 / 执行 / 生活 / 知识 / 觉知）；未配置 API Key 时，本地关键词引擎自动兜底。
- **财务条目 — AI 分类建议**：打开任意交易详情，AI 读取标题、备注与金额，从分类目录中推荐最匹配的分类（如"订阅"、"餐饮"、"交通"）。点击紫色 chip 即可一键采用。
- **任务元数据 — AI 自动填充**：打开任务后自动推断分类、标签和所属项目，随时可点击魔法棒按钮重新触发。
- **人脉记录**：AI 为每个联系人生成互动策略、重要性评分（1–5 分）和下一步行动建议。
- **笔记与觉知条目**：AI 可为任意笔记或觉知记录生成摘要报告。
- **仪表盘 — AI Mentor 面板**：基于近期记录，仪表盘呈现模式识别与周期性引导建议。侧边栏 AI 浮球现在始终可见——未配置时引导你去设置，已配置时直接打开对话窗口。

#### 执行模块 — 项目体验升级

- **项目行重构**：每个项目行左侧显示两个彩色标签——`[短期 / 中期 / 长期]` 时间跨度和 `[未开始 / 进行中 / 已结项 / 已暂停]` 状态——编辑和删除操作整齐排列在右侧。
- **项目状态新增"已暂停"**：`ProjectStatus` 新增 `.paused` case，项目可以标记为暂停而无需删除。Projects 面板统计 pill 实时反映真实计数。
- **项目字段升级为 Combobox**：任务详情中的项目字段现在是统一的组合框——可直接输入新项目名，也可点击 `⌃⌄` 从现有项目中下拉选择。AI 根据任务内容自动填入建议项目。

#### AI 可靠性修复

- **模块分类健壮性**：`classifyText()` 最大 token 数从 20 提升至 60，响应解析改为大小写不敏感扫描——"lifestyle"、"It's **Lifestyle**." 等格式均能正确识别。
- **离线关键词扩充**：本地兜底引擎 Lifestyle 条目新增"subscription / membership / fee / 扣费 / 会员 / 订阅 / 续费"等 15 个关键词，即使不配置 API Key，常见消费类捕捉也能正确路由。
- **财务 AI 验证**：AI 建议的分类会在显示前与已知目录进行校验；模糊匹配（如"订阅服务" → "订阅"）可被优雅处理。

#### 存储与设置

- **API Token 改为本地文件存储**（原 Keychain）：Token 存储于 `~/Library/Application Support/NexaLife/ai_api_token.txt`，首次启动自动从 Keychain 迁移。文件路径在 设置 → AI 中可见。
- **Cherry Studio 风格模型选择器**：双栏 Sheet（左侧提供商列表，右侧模型列表）替代原下拉菜单。自定义提供商可通过 Terminal Preferences 风格的 `+`/`−` 按钮添加和删除。

#### 引导流程（Onboarding）

- 新增 **AI Mentor 步骤**——首次启动时即可配置 API Key，或选择跳过稍后在设置中配置。
- 欢迎页面直接提供语言切换菜单（自动 / 中 / EN）。
- 三步进度指示器：选择模式 → 创建档案 → AI Mentor。

#### 架构优化

- `ContentView` 从 620 行精简至 127 行，模块路由提取至 `ModuleWorkspaceRouter`。
- `VitalsView` 从 997 行精简至 481 行，`ExecutionView` 从 1431 行精简至 710 行。
- `WorkspaceTheme` 设计系统统一，所有模块视图共用同一 token 集合。
- Docs 目录移出 app bundle，应用体积从 ~30 MB 缩减至 ~1.9 MB。
- Xcode 构建阶段脚本在每次构建前自动清除 `com.apple.quarantine` xattr，彻底解决反复出现的 CodeSign 失败问题。
