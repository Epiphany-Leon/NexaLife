# NexaLife v0.1.0 发布准备（草稿）

## 1) GitHub Release Notes 草稿

Title:

`v0.1.0`

Description:

```markdown
## NexaLife v0.1.0

首个公开开源版本，提供 NexaLife 的模块化个人管理基础能力。

- 模块化信息管理：Inbox / Execution / Lifestyle / Vitals / Knowledge / Dashboard

### 📦 Assets

- Source code (zip/tar.gz) 由 GitHub 自动生成

欢迎通过 Issues 提交问题与建议。
```

## 2) Tag 与发布命令清单

> 在仓库根目录执行当前仓库目录。

### A. 发布前检查

```bash
git status
rg -n --hidden --glob '!.git' -e 'AKIA[0-9A-Z]{16}' -e 'AIza[0-9A-Za-z_-]{35}' -e 'sk-[A-Za-z0-9]{20,}' -e 'ghp_[A-Za-z0-9]{36}' -e 'xox[baprs]-' -e '-----BEGIN (RSA|DSA|EC|OPENSSH|PRIVATE) KEY-----' .
```

### B. 提交开源准备改动

```bash
git add README.md LICENSE CHANGELOG.md CONTRIBUTING.md CODE_OF_CONDUCT.md .gitignore
git commit -m "chore: prepare open-source baseline (GPL-3.0 + docs)"
```

### C. 推送主分支

```bash
git push origin main
```

### D. 打首个版本标签

```bash
git tag -a v0.1.0 -m "NexaLife v0.1.0"
git push origin v0.1.0
```

### E1. 使用 GitHub CLI 创建 Release（推荐）

```bash
# 可选：先确认 gh 已登录
gh auth status

# 如果你有本地产物，放到当前目录后附加 --verify-tag + 文件路径
gh release create v0.1.0 \
  --title "v0.1.0" \
  --notes-file NexaLife/Docs/release/v0.1.0/v0.1.0-release-notes.md \
  --verify-tag
```

如果有二进制产物和校验文件：

```bash
gh release create v0.1.0 \
  --title "v0.1.0" \
  --notes-file NexaLife/Docs/release/v0.1.0/v0.1.0-release-notes.md \
  --verify-tag \
  ./NexaLife-macos-v0.1.0.zip \
  ./SHA256SUMS.txt
```

### E2. 使用 Web 页面发布（无需 CLI）

1. 进入仓库 `Releases` -> `Draft a new release`
2. 选择 tag `v0.1.0`
3. Title 填 `v0.1.0`
4. 粘贴本文件第 1 节的说明文本
5. 上传可选资产（zip / SHA256SUMS）
6. 点击 `Publish release`
