## 更新记录 1：Lifestyle 详情页对齐与目标打卡流程重构

### 目标与问题
1. 账户详情页右侧内容未统一对齐，昵称/预算输入区位置不够规整。
2. 目标进度打卡入口分散，缺少结构化打卡弹窗与分钟级记录。
3. 目标状态切换存在误触后不可恢复的问题。
4. 日期展示格式不统一，需要统一为 `yyyy-MM-dd`。

### 关键实现
- 账户详情页（`Views/Settings/AccountDetailView.swift`）
  - 新增统一右对齐行封装 `rightAlignedRow(...)`，将昵称、月度预算、货币、Token、路径等内容统一贴齐右侧边界并保留安全边距。
  - 昵称与月度预算编辑框移动到对应 block 最右侧。
  - 头像操作按钮区改为右对齐，和其余行保持同一视觉轴线。
  - 弹窗默认宽度由 `720` 调整为 `640`。

- 目标详情页（`Views/Lifestyle/GoalDetailView.swift`）
  - 将“记录当前进度”改为“打卡”按钮，新增 `GoalCheckInSheet` 弹窗。
  - 打卡弹窗字段：
    - 打卡标题
    - 打卡日期（图形日历）
    - 打卡时间（精确到分钟）
    - 本次进度（滑块 + 快速百分比）
    - 详情备注
    - 可选“本次打卡后直接设为已完成”
  - “开始时间”显示与编辑调整为仅日期（不展示小时分钟），通过图形日历选择。
  - “截止日期”展示改为日期文本 + 日历开关选择。
  - 状态切换保护：
    - 置为已完成时保存完成前进度快照。
    - 从已完成恢复时优先回滚到快照，避免误触丢失历史进度。
  - 周期 Tracking 区提示语同步更新为“点击打卡开始记录”。

- 打卡数据模型（`Models/GoalProgressEntry.swift`）
  - 新增字段 `title`，用于保存打卡标题并在“最近记录”列表展示。

- 日期格式统一
  - 目标详情页内日期展示统一为 `yyyy-MM-dd`（含开始/截止/最近打卡记录）。
  - 打卡弹窗内增加日期展示文本，保持同一格式。

## 更新记录 2：Dashboard Lifestyle 卡片增加人脉统计 + 收件箱联动复核

### Dashboard 增强
- 文件：`Views/Dashboard/DashboardView.swift`
- Lifestyle 卡片新增人脉维度统计：
  - 人脉总数
  - 高优先人脉（重要性 >= 4）
  - 待跟进人脉（30 天未联系或未记录）
- 展示文案由：
  - `收入 · 目标`
  - 扩展为 `收入 · 目标 · 人脉（高优先 / 待跟进）`

### 收件箱联动复核结论
- 已打通主链路：
  1. 快速输入支持 AI 分类后直接写入四象限目标模块（Execution/Knowledge/Lifestyle/Vitals）。
  2. Inbox 详情页支持手动转移到四象限，并删除原 Inbox 条目。
  3. `InboxRoutingService` 已覆盖四象限实体创建：
     - Execution -> `TaskItem`
     - Knowledge -> `Note`
     - Lifestyle -> `Goal` / `Connection` / `Transaction`
     - Vitals -> `VitalsEntry`

- 当前边界（待后续增强）：
  1. Lifestyle 转移属于启发式子类型判断，尚未在 Inbox 显式让用户选择“账务/目标/人脉”。
  2. 转移后仅切模块，不会自动定位到新建实体详情。
