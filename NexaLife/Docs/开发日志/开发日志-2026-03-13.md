# 构筑人生 / NexaLife 开发日志（2026-03-13）

## 更新记录 1：产品定位收敛为本地优先的 Profile 模式

### 目标与问题
1. 原先“账户 + 云同步”的方向会天然引入开发者托管数据库，不符合隐私优先目标。
2. 用户需要多端迁移能力，但不一定希望把核心数据交给开发者保存。

### 关键实现
- 将产品策略收敛为：
  - 默认：App 内部存储
  - 标准能力：JSON 快照导入导出
  - Apple 用户：后续可选 iCloud 同步
  - 高级用户：可选外部目录快照同步
- 将面向用户的“账户/账号”概念逐步替换为 `Profile`，用于描述本机身份、导出归属与同步偏好，而非开发者托管的云端账户。

## 更新记录 2：Onboarding 改造为 Profile 创建与邮箱验证流程

### 目标与问题
1. 邮箱创建需要先验证邮箱真实性，才能作为通知与版本更新触达渠道。
2. Apple ID 能力尚未完成签名与 capability 配置，需要避免构建阻塞。

### 关键实现
- Onboarding 新增邮箱验证码发送、验证码输入与验证后继续的流程。
- Apple ID 入口保留为“开发中”状态，避免在当前签名条件下触发构建报错。
- 引导文案统一从 Account 收敛到 Profile，突出本机数据归属与本地优先定位。

## 更新记录 3：Settings 重构为 Profile / General / AI / Data / Shortcuts / Privacy

### 目标与问题
1. Profile 信息此前散落在 Data 页和账户详情中，不便管理。
2. Data 页既承载存储策略又承载身份信息，信息层次混乱。

### 关键实现
- 在 Settings 中增加独立 `Profile` Tab：
  - 管理 Profile 名称
  - 展示 Profile 类型
  - 展示并遮罩 Profile ID
  - 支持悬停显示、点击复制 ID
  - 承接删除 Profile 操作
- Data 页只保留：
  - 同步模式选择（本机 / iCloud / 外部目录）
  - 本地存储信息
  - 导入导出
  - 备份、校验、修复与清空内容

## 更新记录 4：About 窗口与品牌命名完成首轮定稿

### 目标与问题
1. General 页不应承载本应属于 About 的品牌介绍与产品说明。
2. 应用需要统一的中英文名称、版本号和卷首语。

### 关键实现

- 版本号定为 `v0.1.1`。
- 品牌名称定为：
  - 中文：`构筑人生`
  - 英文：`NexaLife`
- About 内容新增中英文卷首语、产品策略、数据与隐私说明。

## 更新记录 5：本地化切换与设置页显示问题修复

### 目标与问题
1. Settings 与账户详情中存在中英文切换不完整、显示原始本地化 key 的问题。
2. About 介绍段落在多行文本渲染时缺少稳定空行。

### 关键实现
- 修正 Settings 中对本地化文本的渲染方式，避免字面量 key 直接出现在界面上。
- 账户详情弹窗显式继承当前 app locale，确保语言切换立即生效。
- About 改为显式分段渲染，保证卷首语之间的空行稳定显示。

## 更新记录 6：数据清理与同步边界继续收紧

### 目标与问题
1. 清空数据操作需要真正作用于主窗口使用的同一份数据容器。
2. 外部目录不应继续被误认为主数据库位置。

### 关键实现
- 修正 Settings 场景与主窗口之间的 ModelContainer 绑定，保证清空内容时四象限等数据一起清除。
- 将外部目录定位为“快照同步目标”，不再作为实时主数据库路径。
- 自动备份与导出归档路径继续围绕 App 内部存储展开。
