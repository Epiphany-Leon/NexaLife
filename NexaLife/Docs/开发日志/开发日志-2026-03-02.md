## 更新记录 1：Inbox / Execution / Dashboard / AI / 数据模型

### 目标与问题
1. AI 分类建议滞后、被旧请求覆盖、手动选择后未稳定保存。
2. Dashboard 月切自动存档逻辑与 Execution 的 Done 任务归档联动。
3. Execution 详情能力不足（分类/tag/备注/项目），以及项目管理缺失。
4. Dashboard 展示信息不够直观（Execution 三状态计数、Vitals/Knowledge 标签化展示）。

### 关键实现
- Inbox + AI 分类：
  - `QuickInputSheet` 增加请求版本控制（`aiRequestID`）和防抖，避免旧 AI 返回覆盖新输入。
  - 删除重复“修改分类”入口，保留单一手动分类入口。
  - 手动选择分类时取消待执行 AI 请求，确保用户选择优先。
  - `ItemDetailView` 增加“重新分析”，并优化自动重算触发逻辑。
- Execution：
  - `TaskItem` 新增 `category`、`tagsText`、`completedAt`、`archivedMonthKey`。
  - `TaskDetailView` 支持标题、时间、状态、项目、分类、Tag、截止日期、备注编辑。
  - `AddTaskSheet` 支持分类/tag/项目 AI 建议与自动补齐。
  - 新增 `ExecutionProject`（短/中/长期、名称、说明、更新时间）。
  - `ExecutionView` 增加项目管理入口、项目概览和 `ProjectManagementSheet`。
- Dashboard 自动归档 + 展示：
  - 页面出现与跨天事件触发自动月切归档，无需确认。
  - 同步归档 Execution Done 任务（写 `archivedMonthKey`）。
  - Execution 卡片展示待办/进行中/已完成三状态计数。
  - Knowledge/Vitals 改为胶囊标签展示。
- AI 服务增强：
  - `AIService.classifyText` 增加无 API Key 本地规则兜底。
  - 新增 `suggestTaskMetadata`（分类/标签/项目）并加强 JSON 解析容错。

### 数据层变更
- 新增模型：`ExecutionProject`
- `TaskItem` 扩展字段：`category` / `tagsText` / `completedAt` / `archivedMonthKey`
- `NexaLifeApp` 与 `PreviewHelper` 的 Schema 已加入 `ExecutionProject`

### 需求对应
1. 增删改逻辑放到 Detail 栏。

3. Lifestyle 归类不应误入“人脉”。

6. 统计指标仅使用中文。
7. 账户详情页增加退出键，昵称和预算增加保存键。
8. API Token 支持本地文件直选；默认掩码；眼睛切换明文；明文点击复制。
9. 头像支持裁剪并以圆形呈现。

### 本次实现
- 结构重构（核心）：
  - `ContentView` 新增 Lifestyle 级别状态联动：
    - `selectedLifestyleTab`
    - `selectedTransaction`
    - 复用 `selectedGoal` / `selectedConnection`
  - Lifestyle 详情区改为按 Tab 路由到右侧 Detail：
    - 财务 -> `AccountingTransactionDetailView`
    - 目标 -> `GoalDetailView`
    - 人脉 -> `ConnectionDetailView`
- `LifestyleView`：
  - 改为接收外部 Binding 状态；
  - 切换 tab 时自动清理其他模块选中态；
  - 中栏宽度调整为 `min 360 / ideal 480 / max 620`，改善文字排布。

- 财务（`AccountingView`）
  - 中栏仅保留统计、筛选、列表；编辑删除迁移到右侧 Detail。
  - “记一笔”改为创建草稿条目并选中，具体编辑在右栏完成。
  - 分类筛选改为“下拉按钮 + 可搜索弹窗”（`AccountingCategoryPickerSheet`）。
  - 统计 4 项改为纯中文：`月度预算 / 实际支出 / 收入 / 净收支`。
  - 新增右栏详情 `AccountingTransactionDetailView`：保存/删除/分类/币种/标题/备注/日期完整 CRUD。

  - `ConnectionDetailView` 增加 `selectedConnection` 绑定，删除时清空选中。

  - `InboxRoutingService.looksLikeConnection` 关键词收紧为联系人语义，避免普通消费文本误入“人脉”。

  - 顶部新增“关闭”按钮。
  - 昵称新增“保存”按钮；月度预算新增“保存”按钮。
  - API Token：
    - 默认掩码显示；
    - 眼睛按钮切换明文/掩码；
    - 明文状态下点击 token 即复制到剪贴板并提示“已复制”。
  - 保存位置为本地文件时，新增“选择文件…”（`NSSavePanel`）直接指定 token 文件路径。
