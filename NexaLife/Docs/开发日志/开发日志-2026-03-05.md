## 更新记录 1：Knowledge 重构（中栏筛选 + Detail 栏 CRUD + Markdown 编辑）

### 目标与问题
1. Knowledge 的增删改需要统一收敛到 Detail 栏。
2. 除搜索外，需要支持按“主题 tag”筛选，且 tag 多时可横向查看和列表选择。
3. 取消副标题输入，正文支持 Markdown 编辑/预览/分栏与 `.md` 导入。
4. 统计区样式需要与其余象限一致。

### 关键实现
- `Views/Knowledge/KnowledgeView.swift`
  - 中栏支持主题多选筛选（Tag 条 + 列表筛选弹窗）。
  - 主题统计、已分类/未分类、本周更新等卡片化展示。
  - 列表分组展示主题（未筛选时按主题分栏）。
- `Views/Knowledge/NoteDetailView.swift`
  - 新建/编辑/删除逻辑放在 Detail 栏。
  - 副标题移除。
  - Markdown 编辑区支持 `编辑 / 预览 / 分栏` 模式。
  - 增加 H1/H2/H3、粗体、斜体、列表、引用、代码、链接快捷插入。
  - 支持导入 `.md` 文件并写入正文。
- `Utilities/KnowledgeTagCodec.swift`
  - 统一主题 tag 的解析与回写格式，兼容多 tag 场景。

## 更新记录 4：日期格式统一与中栏条目宽度统一

### 目标与问题
1. 所有日期输入/选择区域需要统一展示为 `yyyy-MM-dd`。
2. 各象限中栏列表条目宽度与左右留白不一致，需要统一。

### 关键实现
- 日期格式统一
  - `Services/CurrencyService.swift` 中新增 `AppDateFormatter.ymd(_:)`。
  - 应用到：
    - Execution：任务列表截止日期、新建任务与任务详情的日期选择展示。
    - Lifestyle：账务列表日期与账务详情日期选择展示、联系人日期选择展示。
    - Vitals：中栏日期与详情头部日期、存档副标题日期。
  - Goal 模块原有 `yyyy-MM-dd` 实现保持不变。
- 中栏条目宽度统一
  - 在各模块 RowView 增加 `.frame(maxWidth: .infinity, alignment: .leading)`。
  - 在各模块中栏 `List` 行增加统一 `listRowInsets(EdgeInsets(top: 4, leading: 12, bottom: 4, trailing: 12))`。
  - 覆盖模块：Dashboard 存档、Inbox、Execution、Knowledge、Lifestyle（账务/目标/人脉）、Vitals。
